# Pre-Guardrails Question Template

用途：这组问题来自 pre-guardrails 历史 trace，可用于最终汇报中复现早期问题，或作为后续回归测试/人工演示模板。

建议使用方式：

1. 在当前版本重置 demo 数据后，按顺序输入这些问题。
2. 观察系统是否仍会出现旧问题，例如地点被改写成“镇北枯井”、状态摘要不可信、开始钥匙任务和完成钥匙任务混淆。
3. 与 `data/history/` 中的 pre-guardrails trace 对比，说明系统迭代前后的差异。

## 旧版暴露的问题

- 真实 LLM 曾把地下遗迹入口改写为“镇子北边的枯井”。
- 旧版 decision 中的 `state_summary` 由 LLM 返回，可能与实际工具执行顺序不一致。
- “询问钥匙细节”和“归还钥匙”曾被粗略归到同一个 `complete_lost_key_quest` 意图。
- 历史 trace 中旧 schema 没有 `state_before` / `state_after`。
