# Optional LLM Integration

The project runs in `mock` mode by default. This keeps the MVP runnable without API keys.

## Default Mode

```bash
streamlit run app.py
```

Default provider:

```text
AGENT_NPC_LLM_PROVIDER=mock
```

In this mode, `src/agent/decision.py` uses deterministic structured decisions.

## OpenAI-Compatible Mode

Recommended local setup:

1. Copy `.env.example` to `.env`.
2. Replace `AGENT_NPC_LLM_API_KEY` with your real API key.
3. Adjust `AGENT_NPC_LLM_MODEL` and `AGENT_NPC_LLM_BASE_URL` for your provider.
4. Restart Streamlit.

DeepSeek example:

```text
AGENT_NPC_LLM_PROVIDER=openai_compatible
AGENT_NPC_LLM_API_KEY=your_api_key
AGENT_NPC_LLM_MODEL=deepseek-chat
AGENT_NPC_LLM_BASE_URL=https://api.deepseek.com
AGENT_NPC_LLM_TIMEOUT=30
AGENT_NPC_LLM_RETRIES=1
AGENT_NPC_MEMORY_LLM_ENABLED=1
```

You can also set environment variables directly:

```bash
set AGENT_NPC_LLM_PROVIDER=openai_compatible
set AGENT_NPC_LLM_API_KEY=your_api_key
set AGENT_NPC_LLM_MODEL=gpt-4o-mini
set AGENT_NPC_LLM_BASE_URL=https://api.openai.com/v1
streamlit run app.py
```

PowerShell example:

```powershell
$env:AGENT_NPC_LLM_PROVIDER = "openai_compatible"
$env:AGENT_NPC_LLM_API_KEY = "your_api_key"
$env:AGENT_NPC_LLM_MODEL = "gpt-4o-mini"
$env:AGENT_NPC_LLM_BASE_URL = "https://api.openai.com/v1"
streamlit run app.py
```

## Structured Decision Contract

The real LLM must return JSON with:

```text
intent
reasoning
memory_policy
response_style
response_keywords
tools
```

The LLM must not return or guess state snapshots. `state_before` and `state_after` are generated by the workflow from SQLite, so trust, affection, quest status, inventory, and unlocked locations remain system-owned facts.

The LLM also does not own long-term memory writes. It can describe the current decision's memory relevance in the `memory_policy` text field, but actual long-term memory creation is handled after tool execution by `src/agent/memory_policy.py`.

Supported intents:

- `start_lost_key_quest`
- `complete_lost_key_quest`
- `reveal_ruins_entrance`
- `withhold_ruins_entrance`
- `general_conversation`

Allowed tools:

- `update_trust`
- `update_affection`
- `give_item`
- `update_quest_status`
- `unlock_location`
- `record_world_event`

## Safety Fallback

If the decision LLM call fails, returns invalid JSON, or violates business rules, the system falls back to mock decision logic and records an `llm_fallback` field in the decision trace. This prevents classroom demos from failing because of API/network problems.

Business rules are intentionally narrow. They constrain only major state and task consistency:

- `start_lost_key_quest` must set `lost_key` to `in_progress` and must not unlock the ruins.
- `complete_lost_key_quest` must set `lost_key` to `completed` and must not unlock the ruins directly.
- `reveal_ruins_entrance` must unlock `underground_ruins_entrance`.
- `withhold_ruins_entrance` must not unlock any location.

## Response Polishing

After tools execute and SQLite state has been updated, `src/agent/response.py` can call the same OpenAI-compatible provider to polish Lina's final reply.

The response LLM receives:

- player input;
- intent and reasoning;
- `response_style`;
- `response_keywords`;
- latest NPC, player, and quest state;
- canonical world facts;
- retrieved memories;
- tool calls and state changes.

Memory Policy runs after response generation, so it is not part of the response LLM input.

It must return:

```json
{
  "npc_response": "Lina 的最终中文回复"
}
```

If this step fails or contradicts a major canonical fact, the system falls back to a deterministic template response and records `response_generation.mode = "fallback_template"` in the decision trace. The guard is deliberately limited: Lina may still add gestures, tone, hesitation, and small tavern atmosphere details as long as she does not change major facts such as the ruins entrance location.

## Memory Candidate And Review LLM

When `AGENT_NPC_LLM_PROVIDER=openai_compatible` and `AGENT_NPC_MEMORY_LLM_ENABLED=1`, Memory Policy uses the same API configuration for two additional LLM calls:

```text
src/agent/llm_memory_candidate.py
-> generate memory candidates

src/agent/memory_candidate_review.py
-> review and approve/reject/revise candidates
```

These modules use the same:

```text
AGENT_NPC_LLM_API_KEY
AGENT_NPC_LLM_MODEL
AGENT_NPC_LLM_BASE_URL
AGENT_NPC_LLM_TIMEOUT
AGENT_NPC_LLM_RETRIES
```

They still do not write SQLite directly. `src/agent/memory_candidate_gate.py` checks allowed memory type, evidence support, state/tool support, player-grounded profile/preference memories, and deduplication before `memory_policy.py` writes approved memories.
